# IAM (Identity and Access Management)

---

## IAM 정책 기본 원칙

- 정책은 **Effect(효과)**, **Action(액션)**, **Resource(리소스)** 등으로 구성된 정책 문서로 정의
- 허용(Allow) 또는 거부(Deny)를 명시적으로 선언
- **명시적으로 허용되지 않은 모든 동작은 기본적으로 거부(Implicit Deny)**
- 명시적 Deny는 Allow보다 항상 우선

## 루트 계정 보안

| 항목 | 내용 |
|------|------|
| 액세스 키 | 루트 계정에 연결된 모든 액세스 키 삭제 |
| 패스워드 | 길고 복잡한 패스워드 설정 |
| MFA | 루트 계정에 다중 인증(MFA) 활성화 |
| 일상 작업 | 루트 계정은 일상적인 관리 작업에 사용 자제 |

---

## IAM 액세스 키 관리

- **미사용 키 비활성화**: 사용하지 않는 액세스 키는 즉시 비활성화
- **키 로테이션**: 주기적으로 액세스 키를 교체하여 보안 강화

---

## IAM 그룹

- 사용자를 팀/역할 단위로 묶어 그룹에 권한 부여
- 구성원 변경 시 그룹 정책만 수정하면 **일괄 적용** 가능
- 권한 관리의 일관성 및 효율성 확보

---

## IAM 롤 (Role)

- 사용자 또는 서비스가 리소스 접근 시 사용하는 **임시 신분(Temporary Identity)**
- 영구 자격증명 없이 단기 토큰으로 권한 부여
- 주요 활용 사례:
  - **Amazon 서비스** (EC2, Lambda 등)
  - **Amazon Cognito** (모바일/웹 앱 사용자)
  - **Google / SAML 기반 외부 IdP** (연동 로그인)

---

## 접근 권한 관리 서비스

### Amazon Cognito

모바일 및 웹 앱 개발자를 위한 인증·권한 서비스

| 기능 | 설명 |
|------|------|
| **User Pool (유저 풀)** | 앱에 회원가입 및 로그인 기능 추가 |
| **Identity Pool (아이덴티티 풀)** | 앱 사용자에게 AWS 서비스에 대한 임시 접근 권한 부여 |

### AWS Managed Microsoft AD

- 온프레미스 **Active Directory**를 AWS에서 관리형으로 제공
- 기존 AD 기반 인증 인프라와 통합 가능

### AWS Single Sign-On (SSO)

- AWS Directory Service로 관리되는 신원 확인 및 권한 부여를 **일관되게 수행**
- 여러 AWS 계정 및 애플리케이션에 단일 로그인으로 접근

---

## 암호화 및 키 관리 서비스

| 서비스 | 설명 |
|--------|------|
| **AWS KMS** (Key Management Service) | 암호화 키 생성 및 관리, 완전관리형·중앙제어형 |
| **AWS Secrets Manager** | DB 자격증명, API 키 등 민감 정보를 안전하게 저장·자동 로테이션 |
| **AWS CloudHSM** | 전용 하드웨어 보안 모듈(HSM)로 키를 직접 제어, 규정 준수용 |

## AWS Resource Access Manager (RAM)

- AWS 계정 간 또는 조직 내에서 **리소스를 안전하게 공유**
- 별도 복제 없이 VPC 서브넷, Transit Gateway, License Manager 구성 등 공유 가능
