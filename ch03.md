# AWS 스토리지 서비스

## Amazon S3 (Simple Storage Service)

### 기본 특성
- 하나의 계정당 최대 **100개의 버킷** 생성 가능
- **단일 AWS 리전에만 존재** (단, 버킷끼리 복제는 가능)
- HTTP 접근 URL 지원
- CLI 접근 URL 지원

### 주요 기능

#### 1. 멀티 파트 업로드
- 대용량 파일 전송에 사용
- 네트워크 중단 시 재시작 가능

#### 2. 암호화
- **KMS(Key Management Service)** 기반 암호화 지원

#### 3. 로그 관리
S3 서버 접근 로그는 다음 정보를 기록:
1. 요청자의 계정 및 IP 주소
2. 소스 버킷 이름
3. 요청 작업 내역
4. 요청을 한 시간
5. 요청에 대한 응답 상태

#### 4. 내구성 (Durability)
- 연간 데이터가 **손실되지 않고 보존될 수 있는 능력**을 퍼센트로 나타냄
- 예: 99.999999999% (11개의 9)

#### 5. 가용성 (Availability)
- 연간 객체에 대한 **지속적인 요청을 처리할 수 있는 능력**을 퍼센트로 표시
- 예: 99.99%

#### 6. 종국적 일관성 (Eventual Consistency)
- S3는 다수의 지역에 데이터를 복제
- 모든 복제본이 동기화되는 데 약간의 시간 소요

#### 7. 생애주기 및 버전관리
- 객체의 생애주기 정책 설정 가능
- 버전 관리를 통한 객체 이력 관리

#### 8. 접근제어
- **IAM 정책** 기반 접근 제어
- **S3 Access Points** 사용 가능

#### 9. 프리사인 URL (Presigned URL)
- 기본 만료 시간: **1시간** (권장)
- 임시 접근 권한 제공
- 정적 웹사이트 호스팅 시에도 사용됨

#### 10. 정적 웹사이트 호스팅
- S3 버킷을 정적 웹사이트로 호스팅 가능
- 프리사인 URL 활용

## Amazon S3 Glacier

### 특징
- S3의 스토리지 클래스 중 하나
- **저렴한 비용**으로 대용량 데이터 장기 보관
- 자주 접근하지 않는 데이터 보관에 적합
- 데이터 검색 시 시간 소요 (Retrieval)

---

## Amazon EFS (Elastic File System)

### 특징
- **리눅스 인스턴스**를 위한 파일 스토리지
- 높은 **확장성**과 **공유성**
- 여러 EC2 인스턴스에서 동시 접근 가능

---

## Amazon FSx

### 특징
- **고성능 파일 시스템**에 접속 가능
- 오픈소스 **분산 파일 시스템** 지원
- Windows/Linux 환경 모두 지원

---

## AWS Storage Gateway

### 특징
- **로컬 스토리지**와 **클라우드 스토리지** 연결
- 온프레미스 환경과 AWS의 연결 역할
- 하이브리드 클라우드 구성에 적합

---

## AWS Snow Family

### 특징
- **대규모 데이터 이전** 시 사용
- 네트워크 기반 전송이 비효율적일 때 활용
- 물리적 장치를 통한 데이터 전송

### 제품군
- **AWS Snowcone**: 소규모 데이터 전송
- **AWS Snowball**: 중규모 데이터 전송
- **AWS Snowmobile**: 초대규모 데이터 전송

---

## AWS DataSync

### 특징
- **온프레미스 데이터**를 **AWS 계정으로 이동**하는 작업에 특화
- 자동화된 데이터 전송
- 검증 및 모니터링 기능 포함

---

## 서비스 선택 가이드

| 용도 | 서비스 | 특징 |
|------|--------|------|
| 일반적인 객체 저장 | S3 | 확장성, 내구성, 가용성 우수 |
| 장기 보관/저렴한 비용 | S3 Glacier | 저비용, 느린 접근 |
| 공유 파일 스토리지 (Linux) | EFS | 높은 공유성, 확장성 |
| 고성능 파일 시스템 | FSx | 고성능, 분산 시스템 |
| 하이브리드 클라우드 | Storage Gateway | 온프레미스-클라우드 연결 |
| 대규모 데이터 마이그레이션 | Snow Family | 물리적 전송 |
| 온프레미스→AWS 데이터 동기화 | DataSync | 자동화된 마이그레이션 |

## 핵심 개념 정리

### 내구성 vs 가용성
- **내구성**: 데이터 손실 방지 능력 (저장된 데이터가 얼마나 안전한가?)
- **가용성**: 서비스 접근 가능성 (얼마나 자주 접근할 수 있는가?)

### S3 접근 방식
1. **HTTP URL**: 웹 브라우저에서 직접 접근
2. **CLI URL**: AWS CLI를 통한 명령어 접근
3. **프리사인 URL**: 임시 접근 권한을 가진 URL

### 데이터 일관성
- S3는 **종국적 일관성(Eventual Consistency)** 모델 사용
- 복제 완료 시까지 약간의 지연 발생 가능
