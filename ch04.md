# VPC (Virtual Private Cloud)
 
## VPC란?
- AWS 클라우드 내에서 논리적으로 격리된 네트워크 공간
- 자신의 데이터 센터를 AWS에서 운영하는 것처럼 제어
- 리전 단위로 생성되며, 여러 가용성 영역(AZ)에 걸쳐 확장 가능
### VPC의 특징
- **격리성**: 다른 VPC와 완벽히 격리
- **커스터마이징**: IP 범위, 서브넷, 라우팅 테이블 자유롭게 설정
- **확장성**: 리소스 추가/제거 용이
- **다양한 연결**: 온프레미스, 다른 VPC와 연결 가능

## VPC 구성 요소
 
### CIDR 블록 (Classless Inter-Domain Routing)
```
VPC CIDR: 10.0.0.0/16  (65,536개 IP 주소)
- 10.0.0.0 ~ 10.0.255.255
 
서브넷 CIDR: 10.0.1.0/24  (256개 IP 주소)
- 10.0.1.0 ~ 10.0.1.255
```
 
#### CIDR 범위
- `/16`: 65,536개 IP (VPC에 권장)
- `/24`: 256개 IP (일반 서브넷)
- `/32`: 1개 IP (호스트)
- `/0`: 0.0.0.0/0 (모든 IP)
#### AWS 예약 IP (5개)
- .0: 네트워크 주소
- .1: VPC 라우터
- .2: DNS 서버
- .3: 미래 사용 예약
- .255: 브로드캐스트 주소
### 서브넷 (Subnet)
- VPC를 여러 개의 작은 네트워크로 분할
- **AZ 단위로 생성** (한 AZ에만 속함)
- 공개 서브넷, 프라이빗 서브넷으로 구분
#### 공개 서브넷 (Public Subnet)
- 인터넷 게이트웨이와 연결
- 인터넷 접근 가능
- EC2 인스턴스에 공인 IP/Elastic IP 할당 가능
#### 프라이빗 서브넷 (Private Subnet)
- 인터넷 접근 불가 (기본)
- NAT 게이트웨이/인스턴스로 아웃바운드 가능
- 데이터베이스, 백엔드 서버 배치
