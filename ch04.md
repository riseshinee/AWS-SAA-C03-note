# AWS VPC (Virtual Private Cloud)

## 1. VPC 기본 개념

### VPC란?
- **정의**: 여러 AWS 서비스에 네트워크 리소스를 담을 수 있는 가상 네트워크
- **범위**: 한 리전 안에서만 존재
- **특징**: AWS 리소스들을 논리적으로 격리된 네트워크 환경에 배치

---

## 2. VPC CIDR 블록

### CIDR란?
- **CIDR (Classless Inter-Domain Routing)**: IP 주소 할당 방법

### CIDR 블록의 역할
- VPC 내의 인스턴스 및 리소스에 할당되는 IP 주소를 결정
- 기본 CIDR 블록은 반드시 할당되어야 함

### CIDR 표기법과 프리픽스
| 프리픽스 | 주소 개수 | 설명 |
|---------|---------|------|
| /16 | 65,536개 | 대규모 네트워크 |
| /24 | 256개 | 중규모 네트워크 |
| /28 | 16개 | 소규모 네트워크 |

- **유효한 IPv4 프리픽스 길이**: /0 ~ /32
- **보조 CIDR 블록**: 지정 가능 (추가 IP 주소 범위)
- **IPv6 CIDR**: 지정 불가능

---

## 3. 서브넷 (Subnet)

### 서브넷의 정의 및 역할
- **정의**: VPC에 있는 네트워크 로직 컨테이너
- **목적**: EC2 인스턴스와 같은 VPC 리소스를 연결
- **기능**: 
  - 네트워크 상의 여러 인스턴스를 격리
  - 인스턴스 간의 트래픽 유입 및 유출 제어
  - 인스턴스는 서브넷 내에서 생성됨

### 서브넷 CIDR 블록
- **예약 IP 주소**: 처음 4개 + 마지막 1개 (총 5개) 예약
  - 첫 번째: 네트워크 주소
  - 두 번째: AWS 라우터
  - 세 번째: AWS DNS
  - 네 번째: AWS 예약
  - 마지막: 브로드캐스트 주소
- **제약사항**: 서브넷은 여러 개의 CIDR을 가질 수 없음

### 서브넷의 가용영역 (AZ)
- 서브넷은 하나의 가용영역 내에서만 존재 가능
- 고가용성 구성 시에는 여러 AZ에 서브넷 배치 필요

---

## 4. 네트워크 인터페이스

### ENI (Elastic Network Interface)
- **역할**: 다른 네트워크 리소스와 통신할 수 있도록 함
- **특징**: 인스턴스와 독립적으로 존재 가능
- **용도**: EC2 인스턴스에 네트워크 연결성 제공

### 성능 강화 네트워크
- 고속의 네트워크 처리속도 제공
- 저지연성 제공
- 고성능 애플리케이션에 필수

---

## 5. 인터넷 게이트웨이 (IGW)

### 정의 및 역할
- 인터넷에서 들어오는 요청을 수신할 수 있도록 함
- VPC 리소스와 인터넷 간의 통신을 중개

### 특징
- AWS 리소스 ID를 식별용으로 할당함
- VPC와 1:1로 연결됨
- 필수 구성 요소 (인터넷 접속 필요 시)

---

## 6. 라우트테이블 (Route Table)

### 정의
- VPC 내에서 트래픽의 유입, 유출, 이동을 제어하는 규칙 집합
- 각 서브넷은 라우트테이블과 연결되어야 함

### 라우트 (Route)
- **역할**: 라우트테이블과 연결된 서브넷 내에서의 트래픽 유입 및 유출 결정
- **판단 기준**: 대상 주소 IP 프리픽스에 의해서만 결정

### 라우트 설정 필수 요소
| 항목 | 설명 |
|------|------|
| **대상 주소 IP 프리픽스** | 목적지 IP 범위 (예: 0.0.0.0/0) |
| **타겟 리소스** | 트래픽을 전달할 대상 (IGW, NAT, VPN 등) |

### 라우트 우선순위
- 가장 구체적인 경로(longest prefix match)가 우선됨
- 예: 0.0.0.0/0 보다 10.0.0.0/16이 우선

---

## 7. 보안 그룹 (Security Group)

### 인바운드 규칙 (Inbound Rules)

**역할**: ENI에 유입되는 트래픽의 허용 여부를 결정

**필수 설정 항목**:
| 항목 | 설명 |
|------|------|
| **소스 (Source)** | 트래픽이 오는 곳 (IP 주소, 보안 그룹 등) |
| **프로토콜** | TCP, UDP, ICMP 등 |
| **포트 범위** | 허용할 포트 번호 (예: 80, 443, 22) |

**목적**: 인바운드 규칙을 추가하여 트래픽이 인스턴스로 유입될 수 있도록 함

### 아웃바운드 규칙 (Outbound Rules)

**역할**: 유출되는 트래픽의 허용 여부를 결정

**필수 설정 항목**:
| 항목 | 설명 |
|------|------|
| **대상 주소 (Destination)** | 트래픽이 가는 곳 (IP 주소 범위) |
| **프로토콜** | TCP, UDP, ICMP 등 |
| **포트 범위** | 허용할 포트 번호 |

### 보안 그룹 특징
- **Stateful**: 인바운드 트래픽 허용 시 자동으로 아웃바운드 응답 허용
- **기본 정책**: 기본적으로 인바운드는 모두 차단, 아웃바운드는 모두 허용
